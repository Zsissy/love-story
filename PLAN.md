# 项目规划文档（React + Vite）

## 1. 项目目标
打造一个“我们”的长期纪念网页，分为三个模块：

1. 五年日记：同一天、跨五年纵向记录
2. 恋爱记录：双方可编辑的日常记录区
3. 旅行地图：点亮中国地图坐标并展示照片

## 2. 用户故事（核心）

1. 我可以在五年日记里记录当日想法，并查看过去几年同一天的内容。
2. 我和小猪可以共同编辑恋爱记录，留下生活点滴。
3. 我们可以在地图上标记去过的城市，并点击查看照片。

## 3. 页面结构

### 3.1 单页三模块布局（MVP）

1. 左侧：五年日记模块（纵向时间轴）
2. 中间：恋爱记录模块（可编辑卡片）
3. 右侧：旅行地图模块（中国地图 + 点位）

### 3.2 未来扩展路由

1. `/`：三模块整合页
2. `/diary`：五年日记专页
3. `/love`：恋爱记录专页
4. `/map`：旅行地图专页

## 4. 功能规划

### 4.1 五年日记模块

- 展示方式：纵向时间轴展示最近五年“同一天”
- 交互
  1. 日期选择器（默认今天）
  2. 每年一个文本框，支持即时保存
- 数据结构
  - `id`
  - `date`（YYYY-MM-DD）
  - `year`
  - `content`

### 4.2 恋爱记录模块

- 展示方式：时间轴或卡片流
- 交互
  1. 新增记录
  2. 编辑记录（双方都可编辑）
  3. 删除记录（可选）
- 数据结构
  - `id`
  - `author`（你 / 小猪）
  - `content`
  - `createdAt`

### 4.3 旅行地图模块

- 展示方式：中国地图 + 点位
- 交互
  1. 点亮城市
  2. 点击城市显示照片墙
  3. 支持多张照片
- 数据结构
  - `id`
  - `city`
  - `lat`
  - `lng`
  - `photos[]`
  - `visitedAt`

## 5. 技术方案

- 框架：React + Vite
- 样式：CSS Modules 或 Tailwind（后续可选）
- 地图：ECharts（中国地图支持好）
- 存储（MVP）：localStorage
- 存储（进阶）：Supabase / Firebase（双人同步）

## 6. 组件拆分（建议）

1. `AppLayout`：三栏布局
2. `DiaryTimeline`：五年日记容器
3. `DiaryItem`：单年日记条目
4. `LoveLog`：恋爱记录容器
5. `LoveCard`：恋爱记录卡片
6. `MapPanel`：旅行地图容器
7. `CityModal`：城市照片弹窗
8. `PhotoGrid`：照片墙

## 7. 数据与状态管理

- MVP：本地 `useState + localStorage`
- 后续：抽离 `useDiaryStore / useLoveStore / useMapStore`
- 同步：预留 API 适配层

## 8. 开发顺序（建议）

1. 初始化项目 + 页面布局
2. 五年日记模块（核心体验先落地）
3. 恋爱记录模块（双人编辑逻辑）
4. 旅行地图模块（地图与点位）
5. 视觉风格与动画润色

---

## 假设说明

1. 五年日记默认显示“今天”，并支持选择任意日期
2. 恋爱记录需要标注作者（你 / 小猪）
3. 旅行地图以“城市级点位”为主
